Doublet

Demonstrated knowledge of the A* Search algorithm, d-ary MinHeap, and C++ vectors, maps, and sets

Implemented the code to play the game "Doublet" invented by Lewis Carroll, wherein two words are provided to see if it is possible to transform the starting word to the end word by simply altering one letter at a time, so long as it is found within a word bank provided. The starting word would have every character altered, one at a time, to become all 26 letters in the alphabet, only storing those that were found in the set of words into a vector. A* Search algorithm was then implemented to calculate how far off the word was from becoming the desired end word – this was our heuristic. Each word was stored in a map to keep track of its g-value, h-value, and child nodes in the MinHeap. The programme would print out the number of transformations needed to go from one word to the other.

Cryptography

Demonstrated knowledge of the extended Euclidean algorithm, modular exponentiation, as well as encryption and decryption methods.

The programme would take in ENCRYPT or DECRYPT as a command, along with a filename and key to begin the process. Encryption involved converting each character of the message to become an integer from 0-26 and be strung together to become several groups of long ints. Then, using modular exponentiation, it calculates a C value with the converted long int and outputs it into a given file, each separated by a white space. As for decryption, a key must be calculated with the extended Euclidean algorithm, and calculate for M with this key using the same formula. By converting M back to its long int form, the programme reverses the encoding process by transforming each int to becoming a character once again.
